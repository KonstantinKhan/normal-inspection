# Отчёт: Модуль анализа и исправления стилей ODT-документов

## Цель

Автоматизировать обнаружение и исправление кастомных (автоматических) стилей в ODT-документах, которые отклоняются от
утверждённых корпоративных шаблонов.

## Проблематика

При редактировании ODT-документов текстовые процессоры (LibreOffice, OpenOffice) создают **автоматические стили** —
локальные переопределения, сохраняемые в `<office:automatic-styles>` внутри `content.xml`. Со временем документ
накапливает десятки таких стилей, что приводит к:

- Несогласованному оформлению документов
- Невозможности массового обновления стилей через шаблон
- Сложностям при валидации документов на соответствие ГОСТ/корпоративным стандартам

## Архитектура решения

Модуль `normal-inspection-backend-doc-service` реализован как Kotlin/JVM приложение с использованием библиотеки **ODFDOM
** (org.odftoolkit:odfdom-java:0.13.0) для работы с ODF-документами на уровне XML-структуры.

### Структура модуля

```
com.example.doc.service/
├── OdtStyle.kt          — модели данных (стиль, место использования)
├── StyleDiff.kt          — модели и функции diff-анализа
├── OdtStyleParser.kt     — парсер ODT-документов
├── OdtStyleFixer.kt      — исправление стилей
└── Main.kt               — консольное приложение
```

## Реализованные возможности

### 1. Парсинг структуры ODT-документа

ODT-файл — это ZIP-архив, содержащий XML-файлы. Модуль работает с двумя из них:

- **`content.xml`** — содержимое документа и автоматические стили (`<office:automatic-styles>`)
- **`styles.xml`** — именованные стили (`<office:styles>`) и стили по умолчанию (`<style:default-style>`)

Парсер извлекает:

- Автоматические стили с их свойствами (text-properties, paragraph-properties)
- Именованные стили с указанием parent-style-name и display-name
- Стили по умолчанию для каждого семейства (paragraph, text, graphic)

### 2. Diff-анализ стилей

Для каждого автоматического стиля выполняется сравнение с его прямым родительским стилем:

- **`[changed]`** — свойство переопределено (значение отличается от родительского)
- **`[added]`** — свойство добавлено (отсутствует в родительском)
- Наследованные свойства (не изменённые) не выводятся для компактности

Фильтрация скрипт-специфичных свойств: для документов на кириллице/латинице свойства с суффиксами `-asian` и
`-complex` (CJK/CTL скрипты) скрываются из вывода, так как дублируют базовые.

### 3. Обнаружение мест использования

Для каждого автоматического стиля определяются места в документе, где он применяется:

- Блочные элементы: `<text:p>`, `<text:h>` (абзацы, заголовки)
- Инлайн-элементы: `<text:span>` (фрагменты текста внутри абзаца)

Выводится порядковый номер абзаца и текстовое содержимое.

### 4. Автоматическое исправление

Реализован пайплайн исправления:

1. Замена `text:style-name` во всех абзацах/заголовках, использующих автоматические стили, на целевой стиль (по
   умолчанию `Text_20_body`)
2. Удаление ставших неиспользуемыми автоматических стилей из `content.xml`
3. Сохранение результата как `<имя>_fixed.odt` рядом с оригиналом

Inline-стили (`text:span`) не затрагиваются — исправляются только параграфные стили.

### 5. Консольный интерфейс

Интерактивный пайплайн:

1. Выбор ODT-файла из ресурсов
2. Вывод анализа: стили → места использования → diff с родителем
3. Опция исправления: `[y/n]`
4. Сохранение исправленного файла

Пример вывода:

```
=== Style: P1 (paragraph) ===
Parent: Text_20_body [Text body] (paragraph)

Used in:
  p 2: "Тест, с кастомизацией"

  Text properties (1 added):
    font-size  14pt
  Paragraph properties: no changes

Fix custom paragraph styles? (target: Text_20_body) [y/n]: y

Fixed 1 paragraph(s), removed 1 unused style(s).
Saved to: .../test_fixed.odt
```

## Концепции и технологии

| Концепция                  | Применение                                                                                |
|----------------------------|-------------------------------------------------------------------------------------------|
| ODF (Open Document Format) | Работа со стандартным XML-форматом ODT                                                    |
| XML DOM                    | Навигация по дереву элементов через namespace-aware API                                   |
| ODFDOM                     | Библиотека для чтения/записи ODF-документов                                               |
| Style inheritance          | Разрешение цепочки наследования стилей (default → named → automatic)                      |
| Namespace separation       | Использование URI пространств имён (office, style, text, fo) для точного поиска элементов |
| Diff-анализ                | Сравнение свойств родительского и дочернего стилей                                        |

## Тестовое покрытие

13 модульных тестов в двух классах:

- **OdtStyleParserTest** (9 тестов) — парсинг automatic/common/default стилей, поиск использований, diff-анализ
- **OdtStyleFixerTest** (4 теста) — замена стилей, удаление неиспользуемых, сохранение span-стилей, именование выходного
  файла
